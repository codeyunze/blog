继承是一种机制，子类继承了父类后，可以直接使用父类的方法，无需重复编写。

继承关注的核心为 **“是” 的关系** ，例如猫是动物，狗也是动物，草和树是植物，支付宝和微信支付是支付方式，人民币和美元是货币的问题。在这里面动物、植物、支付方式、货币，就是父类。

```java
class Pay {
    String name;
    
    public void introduction() {
        System.out.println("我是" + name);
    }
    
    public void deductMoney() {
        System.out.println("扣减余额");
    }
}

class Alipay extend Pay {
    @Override
    public void deductMoney() {
        System.out.println("调用支付宝接口扣减余额");
    }
}

class WeChatPay extend Pay {
    @Override
    public void deductMoney() {
        System.out.println("调用微信接口扣减余额");
    }
}

public class Main {
    public static void main(String[] args) {
        Pay pay = new Alipay();
        // 子类可以直接使用父类的属性和方法
        pay.name = "支付宝";
        pay.introduction();
        // 如果子类重写了父类的方法，则调用的就是子类自己的方法
        pay.deductMoney();
    }
}
```



抽象是隐藏具体的实现细节，只暴露行为的设计思想。可以通过抽象类和接口实现，规范其实现类会有哪些行为。

抽象关注的核心为 **“能做”的关系** ，例如猫吃鱼，狗吃骨头；草能加工为草帽，树能加工为木床；支付宝借钱有花呗，微信借钱有微粒贷；人民币可以在中国买东西，美元可以在美国买东西。在这里面，吃、加工、借钱、买东西，就是需要抽象出来的行为。

继承是一个纵向的层级。抽象是横向的层级。

```java 
abstract class Pay {
    String name;
    
    // 介绍
    public void introduction() {
        System.out.println("我是" + name);
    }
    
    // 扣钱
    public void deductMoney();
    
    // 借钱
    public void borrowMoney();
}

class Alipay implement Pay {
    @Override
    public void deductMoney() {
        System.out.println("调用支付宝接口扣减余额");
    }
    
    @Override
    public void borrowMoney() {
        System.out.println("借钱用的花呗");
    }
}

class WeChatPay implement Pay {
    @Override
    public void deductMoney() {
        System.out.println("调用微信接口扣减余额");
    }
    
    @Override
    public void borrowMoney() {
        System.out.println("借钱用的微粒贷");
    }
}

public class Main {
    public static void main(String[] args) {
        Pay pay = new WeChatPay();
        // 子类可以直接使用父类的属性和方法
        pay.name = "微信";
        pay.introduction();
        pay.borrowMoney();
        pay.deductMoney();
    }
}
```
